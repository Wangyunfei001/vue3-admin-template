# Vue3 全家桶最新技术模板架构 需求文档

## 1. 概述

为个人开发准备一个可复用的通用中后台模板仓库，基于 Vue3 生态与现代工程化实践，达到开箱即用、可扩展、可维护的目标。

## 2. 用户场景

- 用户画像：个人开发者（单人维护）。
- 典型场景：新建中后台项目时，希望直接复制模板并快速启动，包含基础页面、权限路由、国际化、主题与暗黑模式、测试与部署能力。
- 当前痛点：每次新项目都要重复搭建工程配置，效率低且容易遗漏质量保障环节。

## 3. 功能需求（P0/P1/P2）

### P0（必须）

- 技术栈基础：`Vue 3`、`Vite`、`TypeScript`、`Pinia`、`Vue Router`。
- 网络层：统一请求封装，默认采用 `axios` 方案实现。
- 权限能力：基础登录态、路由权限示例（至少包含公开路由和受保护路由）。
- i18n：提供中英文切换基础能力与示例文案。
- 主题能力：暗黑模式切换 + 主题变量管理（可扩展）。
- 样式体系：`Tailwind CSS`。
- 多端设备适配：至少覆盖桌面端与常见移动宽度断点的响应式布局基础。
- 模拟后端：提供 mock 数据与接口模拟，不依赖真实后端。
- 工程质量：`lint`、单元测试、端到端测试、`build` 全流程可执行。
- 工程交付：`README` 与目录规范说明。

### P1（应该）

- CI/CD：基础流水线（安装、检查、测试、构建）可在代码托管平台执行。
- Docker：提供开发/生产构建相关基础配置（镜像构建与运行示例）。
- 首页与登录页：提供可运行示例页面并可作为后续业务改造起点。

### P2（可选）

- 更丰富的权限模型示例（如按钮级权限）。
- 更完整的主题系统（多主题动态切换与持久化策略优化）。

## 4. 非功能需求

- 可用性：`pnpm install && pnpm dev` 一次启动成功。
- 可维护性：目录分层清晰、模块职责明确、文档可指导二次开发。
- 兼容性：基于当前主流浏览器环境进行常规兼容（不要求旧版浏览器专项兼容）。
- 质量门禁：`lint/test/build` 全通过作为基础验收前提。

## 5. 范围边界

### 明确包含

- Vue3 全家桶中后台模板的工程搭建、示例页面、mock、测试、部署基础设施。

### 明确不包含

- 不接入真实后端服务，仅使用 mock。
- 不做 SSR（不使用 Nuxt）。
- 不做微前端架构。
- 不包含特定 UI 组件库（当前偏好为“无固定 UI 库”）。

## 6. 约束与依赖

- 包管理器：`pnpm`。
- 版本策略：优先使用“当前最新稳定版”依赖（避免实验版/RC）。
- UI 库偏好：无（以原生组件 + Tailwind 为主，可后续替换）。
- 请求库偏好：`axios`（默认标准实现）。
- Node.js 版本：最低 `20 LTS`。
- 依赖：CI 平台与 Docker 运行环境需满足对应最低要求。

## 7. 风险与缓解

- 依赖“最新版本”带来的兼容波动风险：通过锁定版本与 CI 回归测试缓解。
- 功能面较广导致初版复杂度上升：按 P0 优先交付，P1/P2 逐步补齐。
- 无固定 UI 库可能增加页面开发成本：通过模板内沉淀基础组件与样式规范缓解。

## 8. 验收清单

- [ ] `pnpm install && pnpm dev` 可一次启动成功。
- [ ] 包含登录页、首页、权限路由示例并可运行。
- [ ] 支持 i18n（至少中英文切换）并有示例文案。
- [ ] 支持暗黑模式与主题能力，并可切换验证。
- [ ] 含 mock 接口能力，项目启动后可独立联调示例页面。
- [ ] 响应式在桌面端和常见移动宽度下布局可用。
- [ ] `lint`、单测、端测、`build` 全部通过。
- [ ] 提供清晰 README 与目录规范说明。
- [ ] 提供基础 CI/CD 配置与 Docker 使用说明。

## 9. 决策记录

- 决策：项目定位为“个人可复用的通用中后台模板”。
- 决策：首版只做 mock，不接真实后端。
- 决策：不做 SSR、不做微前端，优先保证 CSR 模板完整度。
- 决策：样式方案采用 Tailwind CSS。
- 决策：默认请求层采用 `axios`。
- 决策：Node.js 最低版本固定为 `20 LTS`。
