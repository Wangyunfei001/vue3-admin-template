# Vue3 全家桶最新技术模板架构实施计划

## 1. 规划目标

- 将已批准需求拆解为可执行任务清单。
- 明确依赖关系、并行策略、风险缓解和里程碑。
- 为下一阶段（`@architect` 或 `/spec`）提供稳定输入。

## 2. 工作分解结构（WBS）

### A. 项目初始化与工程基线（P0）

- A1 初始化 Vue3 + Vite + TypeScript 项目骨架（pnpm）。
- A2 建立目录规范（`src`、`mock`、`tests`、`scripts`、`docs`）。
- A3 集成基础代码规范（ESLint/Prettier/TypeScript strict）。
- A4 配置基础命令（`dev`/`build`/`lint`/`test`/`test:e2e`）。
- A5 固定 Node 与 pnpm 版本约束（Node `20 LTS`）。

### B. 核心能力搭建（P0）

- B1 集成 Router（含公开路由、受保护路由占位）。
- B2 集成 Pinia（用户态、权限态、主题态）。
- B3 集成 axios 网络层封装（实例、拦截器、错误处理）。
- B4 集成 i18n（中英文、语言切换）。
- B5 集成 Tailwind CSS 与基础设计 token。
- B6 集成暗黑模式与主题切换机制（持久化）。

### C. 业务示例与权限流程（P0/P1）

- C1 实现登录页示例（mock 登录流程）。
- C2 实现首页示例（基础统计卡片/欢迎区）。
- C3 实现路由守卫与权限校验（未登录跳转、角色示例）。
- C4 实现 401/403/404 页面与重定向策略。

### D. Mock 与联调能力（P0）

- D1 选型并接入 mock 方案（开发态可用）。
- D2 提供登录、用户信息、菜单/权限等 mock 接口。
- D3 打通前端请求层与 mock 数据链路。
- D4 提供 mock 开关与文档说明。

### E. 多端适配与体验优化（P0）

- E1 定义响应式断点策略（桌面/平板/移动）。
- E2 实现主布局响应式适配（侧栏折叠/抽屉）。
- E3 校验核心页面在常见尺寸可用性。

### F. 测试与质量保障（P0）

- F1 建立单元测试框架与首批用例（store/utils/guard）。
- F2 建立 E2E 测试框架与关键路径用例（登录->首页->退出）。
- F3 接入覆盖率与基础阈值（可后续提升）。
- F4 确保 `lint/test/build` 全通过。

### G. 交付与工程化（P1）

- G1 配置 CI（安装、Lint、Unit、E2E、Build）。
- G2 配置 Docker（开发/生产镜像 + 运行说明）。
- G3 完成 README（启动、脚本、结构、二次开发指南）。
- G4 输出目录规范文档（模块职责与扩展约定）。

## 4. 依赖关系与并行策略

### 主依赖链（关键路径）

`T01 -> T02 -> T04/T05/T06/T07 -> T10 -> T11 -> T13 -> T17 -> T20 -> T21`

### 可并行任务

- 规范与样式：`T03`、`T08` 可与路由/store 基础并行。
- 示例页面：`T12` 与 `T14` 可在 `T11/T13` 后并行推进。
- 工程化：`T18`、`T19` 可在核心功能稳定后并行完成。

## 5. 里程碑与交付物

- M1（基线完成）：T01-T03 完成，可启动、可检查。
- M2（核心可运行）：T04-T11 完成，登录与主流程打通。
- M3（可验收）：T12-T17 完成，功能+测试闭环。
- M4（可交付）：T18-T21 完成，CI/Docker/文档齐备并通过验收。

## 6. 风险与缓解措施


| 风险        | 影响      | 缓解措施                 |
| --------- | ------- | -------------------- |
| 最新依赖兼容波动  | 启动或构建失败 | 锁定稳定版本，分阶段升级，CI 持续验证 |
| 功能面较大导致延期 | 首版不可控   | 严格按 P0 先交付，P1 延后     |
| E2E 环境不稳定 | 测试波动    | 固定浏览器版本，拆分核心路径用例     |
| 响应式复杂度上升  | 体验不一致   | 先确保主流程页面在关键断点可用      |
